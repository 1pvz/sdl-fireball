## Renderer
截止到目前为止，一直使用 `surface` 完成程序，不过接下来需要把 `surface` 换成 `renderer`，因为 `surface` 基本上很难接入其他相关库

`surface` 是软件渲染方案，把图片数据存储内存中，在绘制的时候用 `CPU` 逐个像素绘制，类似 `canvas` 的 `putImageData`

`renderer` 则是 `GPU` 渲染，数据存储在显存中，`GPU` 会一次性处理所有像素，所以会快很多

先在 `game` 结构体里增加 `renderer`

``` c
SDL_Renderer *renderer;
```

创建 `game` 的时候也要创建 `renderer` 并赋值

``` c
SDL_Renderer *renderer = SDL_CreateRenderer(window, NULL);
if (renderer == NULL) {
    SDL_Log("Error creating renderer: %s\n", SDL_GetError());
    SDL_DestroyWindow(window);
    free(g);
    return NULL;
}
g->renderer = renderer;
```

接下来还需要替换 `clear` 和 `draw` 的操作
``` c
SDL_SetRenderDrawColor(game->renderer, 255, 255, 255, 255);
SDL_RenderClear(game->renderer);
```

原本是用 `SDL_BlitSurface` 把像素数据从图片 `surface` 复制到窗口 `surface`

然后用 `SDL_UpdateWindowSurface` 把窗口 `surface` 的数据发给显示器，这样就能在显示器上看到画好的图片

换成 `renderer` 渲染的话，`GPU` 不能直接从内存读取数据，所以需要先把图片数据放在显存中，`GPU` 才能直接读取，也就是从 `surface` 转成 `texture`

复制完 `texture` 之后，需要释放资源，目前是每帧转换数据，实际上应该在渲染之前完成这个事情，但先这么处理

另外 `SDL_RenderTexture` 接受的是 `SDL_FRect`，也就是每个数据都是 `float` 类型

``` c
void
Game_drawImage(Game *self, GuaImage image) {
    SDL_FRect rect = {
        .x = (float)image.x,
        .y = (float)image.y,
        .w = (float)image.image->w,
        .h = (float)image.image->h,
    };
    Game *game = self;
    SDL_Texture *texture = SDL_CreateTextureFromSurface(game->renderer, image.image);
    SDL_RenderTexture(game->renderer, texture, NULL, &rect);
    SDL_DestroyTexture(texture);
}
```

接着在 `Game_draw` 里调用 `SDL_RenderPresent` 把数据画出来
