## Renderer
截止到目前为止，一直使用 `surface` 完成程序，不过接下来需要把 `surface` 换成 `renderer`，因为 `surface` 基本上很难接入其他相关库

`surface` 是软件渲染方案，把图片数据存储内存中，在绘制的时候用 `CPU` 逐个像素绘制，类似 `canvas` 的 `putImageData`

`renderer` 则是 `GPU` 渲染，数据存储在显存中，`GPU` 会一次性处理所有像素，所以会快很多

先在 `game` 结构体里增加 `renderer`

``` c
SDL_Renderer *renderer;
```

创建 `game` 的时候也要创建 `renderer` 并赋值

``` c
SDL_Renderer *renderer = SDL_CreateRenderer(window, NULL);
if (renderer == NULL) {
    SDL_Log("Error creating renderer: %s\n", SDL_GetError());
    SDL_DestroyWindow(window);
    free(g);
    return NULL;
}
g->renderer = renderer;
```

接下来还需要替换 `clear` 和 `draw` 的操作
``` c
SDL_SetRenderDrawColor(game->renderer, 255, 255, 255, 255);
SDL_RenderClear(game->renderer);
```

原本是用 `SDL_BlitSurface` 把像素数据从图片 `surface` 复制到窗口 `surface`

然后用 `SDL_UpdateWindowSurface` 把窗口 `surface` 的数据发给显示器，这样就能在显示器上看到画好的图片

换成 `renderer` 渲染的话，`GPU` 不能直接从内存读取数据，所以需要先把图片数据放在显存中，`GPU` 才能直接读取，也就是从 `surface` 转成 `texture`

复制完 `texture` 之后，需要释放资源，目前是每帧转换数据，实际上应该在渲染之前完成这个事情，但先这么处理

另外 `SDL_RenderTexture` 接受的是 `SDL_FRect`，也就是每个数据都是 `float` 类型

``` c
void
Game_drawImage(Game *self, GuaImage image) {
    SDL_FRect rect = {
        .x = (float)image.x,
        .y = (float)image.y,
        .w = (float)image.image->w,
        .h = (float)image.image->h,
    };
    Game *game = self;
    SDL_Texture *texture = SDL_CreateTextureFromSurface(game->renderer, image.image);
    SDL_RenderTexture(game->renderer, texture, NULL, &rect);
    SDL_DestroyTexture(texture);
}
```

接着在 `Game_draw` 里调用 `SDL_RenderPresent` 把数据画出来

`_Paddle_new` 是 C 的命名风格

不管是 C 还是 C++，编译器会处理程序里的变量名，这个过程叫 **名称修饰(Name Mangling)**

- C 的规则很简单，直接在原始名称的前面增加下划线 `_`
- C++ 的规则会复杂一些
  - `__Z` 是修饰前缀
  - `10` 是原始名称 `Paddle_new` 的长度
  - `v` 表示原始函数的参数为 `void`

所以实际的报错信息是找不到 `__Z10Paddle_newv` 这个变量，但对于开发者来说，这样的名称很困惑，所以编译器（链接器）会显示 `Paddle_new()`，也就是未修饰的原始名称，同时链接器还提示 `found '_Paddle_new' in paddle.o, declaration possibly missing 'extern "C"'`

`extern "C"` 的作用是按照 C 的方式来编译与链接代码，这样 C++ 编译器就不会按照 C++ 的风格来修饰名称

对于 `paddle.h` 来说，直接用 `extern "C"` 来包一层就可以，当然 `extern "C"` 是 C++ 的语法，为了让 C 编译器不报错，把它放在 `__cplusplus` 条件编译宏里面

这样 C 编译里不会看到 `extern "C"` 自然不报错，C++ 编译器看到了就会按照 C 风格处理名称

``` c
#pragma once

#ifdef __cplusplus
extern "C" {
#endif

#include "gua_image.h"

struct _Game;
typedef struct _Game Game;

struct _Paddle;
typedef struct _Paddle Paddle;
struct _Paddle {
    GuaImage image;
    struct _Game *game;
    int speed;
    void (*moveLeft)(Paddle *paddle);
    void (*moveRight)(Paddle *paddle);
};

void
Paddle_move(Paddle *self, int x);

void
Paddle_moveLeft(Paddle *self);

void
Paddle_moveRight(Paddle *self);

Paddle *
Paddle_new(void);

bool
Paddle_collide(Paddle *self, GuaImage image);
#ifdef __cplusplus
}
#endif

```

运行后会出现闪烁异常的情况，这是因为 `Game_draw` 里面会调用 `SDL_RenderPresent`，但现在 `main` 函数里面已经会调用 `SDL_RenderPresent`，这样就会调用两次，所以需要把 `Game_draw` 里面的调用去掉

处理渲染之后需要处理事件绑定，目前没有绑定事件，会导致按键不起作用

原本在 `Game_runLoop` 里面调用 `Game_bindEvents`，但是现在已经不再调用 `Game_runLoop`，那么就需要手动在 `main` 函数里面调用 `Game_bindEvents`

需要注意，`SDL_PollEvent` 会把 `event` 队列的 `event` 取出来，那么 `SDL_PollEvent` 就只能调用一次，换句话说，所有事件都需要在 `SDL_PollEvent` 循环里面处理好

`Game_bindEvents` 函数内部原本会调用 `SDL_PollEvent`，那么需要改造成在 `SDL_PollEvent` 循环里调用 `Game_bindEvents`

``` c
// 在 main 函数 SDL_PollEvent 循环中调用 Game_bindEvents
while (SDL_PollEvent(&event)) {
    ImGui_ImplSDL3_ProcessEvent(&event);
    if (event.type == SDL_EVENT_QUIT) {
        game->quit = true;
    }

    Game_bindEvents(game, &event);
}

// 改造 game.c 的 Game_bindEvents
// game.h 也要同步修改
void
Game_bindEvents(Game *self, SDL_Event *SDLEvent) {
    Game *game = self;
    SDL_Event event = *SDLEvent;
    if (event.type == SDL_EVENT_QUIT) {
        game->quit = true;
    } else if (event.type == SDL_EVENT_KEY_DOWN) {
        SDL_Scancode sc = event.key.scancode;
        if (sc < MAX_COUNT) {
            game->keydowns[sc] = true;
        }
        if (sc == SDL_SCANCODE_P) {
            game->paused = !game->paused;
        }
        unsigned int level = sc - SDL_SCANCODE_1 + 1;
        if (level >= 1 && level <= 4) {
            Game_loadLevel(game, level, game->blockImage);
        }
    } else if (event.type == SDL_EVENT_KEY_UP) {
        SDL_Scancode sc = event.key.scancode;
        if (sc < MAX_COUNT) {
            game->keydowns[sc] = false;
        }
    } else if (event.type == SDL_EVENT_MOUSE_BUTTON_DOWN) {
        float x = event.button.x;
        float y = event.button.y;
        SDL_Point point = {
            .x = x,
            .y = y,
        };
        SDL_Rect rect = {
            .x = game->ball->image.x,
            .y = game->ball->image.y,
            .w = game->ball->image.image->w,
            .h = game->ball->image.image->h,
        };
        if (SDL_PointInRect(&point, &rect)) {
            game->enableDrag = true;
            game->offsetX = x - rect.x;
            game->offsetY = y - rect.y;
        }
    } else if (event.type == SDL_EVENT_MOUSE_BUTTON_UP) {
        game->enableDrag = false;
    } else if (event.type == SDL_EVENT_MOUSE_MOTION) {
        if (game->enableDrag) {
            float x = event.motion.x;
            float y = event.motion.y;
            game->ball->image.x = x - game->offsetX;
            game->ball->image.y = y - game->offsetY;
        }
    }
}
```
