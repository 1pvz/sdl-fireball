## 整理代码结构（三）
### 添加 Ball
目前 `Surface` 上有 `paddle`，现在需要加一个 `ball`，`ball` 的结构和 `paddle` 差不多，有 `image`、`x`、`y` 就可以把 `ball` 画出来，移动还需要 `speed`，所以暂时放这些属性

可以看到这几个属性和 `paddle` 的属性是一样的，但先不用处理，后面再处理

``` c
struct _Ball {
    SDL_Surface *image;
    int x;
    int y;
    int speed;
};
typedef struct _Ball Ball;

Ball *
Ball_new(void) {
    SDL_Surface *image = imageFromPath("ball.png");
    if (image == NULL) {
        return NULL;
    }
    Ball *o = malloc(sizeof(Ball));
    if (o == NULL) {
        SDL_DestroySurface(image);
        return NULL;
    }
    o->image = image;
    o->x = 100;
    o->y = 242;
    o->speed = 10;
    return o;
}
```

同样地，先让 `game` 持有 `ball`

``` c
struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    Paddle *paddle;
    Ball *ball;
    bool keydowns[MAX_COUNT];
    void (*actions[MAX_COUNT])(void *);
    void *data[MAX_COUNT];
};
```

在创建 `game` 的时候，同样需要初始化 `ball`

``` c
g->ball = NULL;
```

接着在 `main` 函数里面调用 `Ball_new` 创建 `ball`，然后挂在 `game` 上面

``` c
Ball *ball = Ball_new();
if (ball == NULL) {
    SDL_Log("Failed to create ball\n");
    SDL_DestroyWindow(game->window);
    free(game);
    SDL_Quit();
    return 1;
}
game->ball = ball;
Game_runLoop(game);
```

画图的时候把 `ball` 画出来

``` c
Ball *ball = game->ball;
SDL_Rect rect1 = {
    .x = ball->x,
    .y = ball->y,
    .w = ball->image->w,
    .h = ball->image->h,
};
SDL_BlitSurface(ball->image, NULL, game->surface, &rect1);
```

可以看到，这部分代码和画 `paddle` 的代码是一样的，所以这里考虑抽出去

画图需要知道 `x`、`y`、`w` 和 `h` 四个属性，那就专门定义一个 `struct` 用来存这几个属性

``` c
struct _GuaImage {
    SDL_Surface *image;
    int x;
    int y;
};
typedef struct _GuaImage GuaImage;
```

接着让这个 `struct` 成为 `Game_drawImage` 的第二个参数

``` c
void
Game_drawImage(Game *self, GuaImage image) {
    SDL_Rect rect = {
        .x = image.x,
        .y = image.y,
        .w = image.image->w,
        .h = image.image->h,
    };
    Game *game = self;
    SDL_BlitSurface(image.image, NULL, game->surface, &rect);
}
```

然后还需要做一件事情，原本 `paddle` 和 `ball` 各自维护了 `x` `y` 这几个属性，现在让它们直接拿到 `image` 就可以

``` c
struct _Paddle {
    GuaImage image;
    int speed;
    void (*moveLeft)(struct _Paddle *paddle);
    void (*moveRight)(struct _Paddle *paddle);
};
```

接着需要把用到 `x` `y` `image` 的地方都要改过来

``` c
void
Paddle_moveLeft(Paddle *self) {
    self->image.x -= self->speed;
}

void
Paddle_moveRight(Paddle *self) {
    self->image.x += self->speed;
}

Paddle *
Paddle_new(void) {
    SDL_Surface *image = imageFromPath("paddle.png");
    if (image == NULL) {
        return NULL;
    }
    Paddle *o = malloc(sizeof(Paddle));
    if (o == NULL) {
        SDL_DestroySurface(image);
        return NULL;
    }
    o->image.image = image;
    o->image.x = 100;
    o->image.y = 250;
    o->speed = 15;
    o->moveLeft = Paddle_moveLeft;
    o->moveRight = Paddle_moveRight;
    return o;
}
```

画图的时候直接调用 `Game_drawImage`
``` c

```

释放资源的时候也要同步修改

### 让 ball 移动
添加 `ball` 之后，`ball` 并不会移动，需要调用 `ball` 的函数
``` c
void
Ball_move(Ball *self) {
    SDL_Log("ball move\n");
}
```

但是这样有一个问题，只定义了函数，没有调用函数，而且希望这个函数每帧都能调用，所以需要放在游戏循环里，现在游戏循环的代码比较多，在放进去之前先抽一些函数出来

游戏的通用逻辑通常如下
- 处理输入
- 更新状态
- 画图

发生 `SDL_EVENT_QUIT` 事件的时候需要结束循环，抽成单独的函数后，把 `quit` 挂在 `game` 上，这样就可以跨函数读取到值

``` c
void
Game_bindEvents(Game *self) {
    Game *game = self;
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            game->quit = true;
        } else if (event.type == SDL_EVENT_KEY_DOWN) {
            SDL_Scancode sc = event.key.scancode;
            if (sc < MAX_COUNT) {
                game->keydowns[sc] = true;
            }
        } else if (event.type == SDL_EVENT_KEY_UP) {
            SDL_Scancode sc = event.key.scancode;
            if (sc < MAX_COUNT) {
                game->keydowns[sc] = false;
            }
        }
    }
}

void
Game_update(Game *self) {
    Game *game = self;
    Ball_move(game->ball);
}

void
Game_draw(Game *self) {
    Game *game = self;
    Paddle *paddle = game->paddle;
    Ball *ball = game->ball;
    Game_drawImage(game, paddle->image);
    Game_drawImage(game, ball->image);
    SDL_UpdateWindowSurface(game->window);
}

void
Game_runLoop(Game *self) {
    Game *game = self;

    const int delay = 1000 / 60;
    while (game->quit == false) {
        Uint32 frameStart = SDL_GetTicks();

        Game_bindEvents(game);
        for (int i = 0; i < MAX_COUNT; i += 1) {
            if (game->actions[i] != NULL && game->keydowns[i]) {
                game->actions[i](game->data[i]);
            }
        }
        // clear
        Uint64 color = SDL_MapSurfaceRGB(game->surface, 255, 255, 255);
        SDL_FillSurfaceRect(game->surface, NULL, color);
        // update
        Game_update(game);
        // draw
        Game_draw(game);

        Uint64 frameEnd = SDL_GetTicks();
        int frameTime = frameEnd - frameStart;
        if (frameTime < delay) {
            SDL_Delay(delay - frameTime);
        }
    }
}
```

注册事件只要在游戏循环开始之前调用就可以，所以完全可以移动到 `main` 函数

现在每一帧都可以调用 `Ball_move` 函数，接下来考虑让 `ball` 真正移动，游戏加载的时候 `ball` 默认不动，按键之后 `ball` 开始移动，可以通过一个 `bool` 属性控制这点

因为 `ball` 会在两个方向移动，所以把 `speed` 换成 `speedX` 和 `speedY` 更合适

按下 `f` 后，会先调用 `fire` 函数，`fire` 函数内部继续调用 `Ball_fire` 函数改变 `ball->fired` 的值

接着在 `Ball_move` 里面就可以改变 `ball` 两个方向的速度，然后再根据速度修改 `ball` 的坐标，因为 `Ball_move` 每一帧都会调用，所以 `ball` 会一直移动

窗口的宽高是直接写死的，实际应该抽出去弄成配置

``` c
void
Ball_fire(Ball *self) {
    self->fired = true;
}

void
Ball_move(Ball *self) {
    Ball *o = self;
    if (o->fired) {
        if (o->image.x < 0 || (o->image.x + o->image.image->w) > 640) {
            o->speedX = -o->speedX;
        }
        if (o->image.y < 0 || (o->image.y + o->image.image->h) > 480) {
            o->speedY = -o->speedY;
        }

        o->image.x += o->speedX;
        o->image.y += o->speedY;
    }
}

void
fire(void *data) {
    Ball *ball = (Ball *)data;
    Ball_fire(ball);
}

Game_registerAction(game, SDL_SCANCODE_F, fire, ball);
```

宽与高在 `Game_new` 里定义的
``` c
const int w = 640;
const int h = 480;
SDL_Window *window = SDL_CreateWindow("Hello SDL3", w, h, 0);
```

把这两个变量抽到最外层，`Game_new` 随着定义随着使用，变量的有效范围特别短，所以用的短变量名 `w` 与 `h`

抽到外层之后就不适合这么短的变量名，直接换成常规名字 `width` 和 `height`，这样整个程序就能直接使用 `width` 和 `height`

另外可以预想到只会在当前文件里使用，所以加上 `static` 修饰，接着把其他需要使用宽高的地方换成 `width` 与 `height`

``` c
static const int width = 640;
static const int height = 480;
```

最后再补上一段 `ball` 撞到 `paddle` 反弹的逻辑，相撞的逻辑先放在 `update` 里面处理

``` c
void
Ball_bounce(Ball *self) {
    self->speedY = -self->speedY;
}

bool
collide(GuaImage a, GuaImage b) {
    // a 在 b 的左边
    bool left = a.x + a.image->w < b.x;
    // a 在 b 的右边
    bool right = b.x + b.image->w < a.x;
    // a 在 b 的上边
    bool up = a.y + a.image->h < b.y;
    // a 在 b 的下边
    bool bottom = b.y + b.image->h < a.y;

    return !(left || right || up || bottom);
}

void
Game_update(Game *self) {
    Game *game = self;
    Ball_move(game->ball);
    if (collide(game->paddle->image, game->ball->image)) {
        Ball_bounce(game->ball);
    }
}
```

既然提取了工具函数 `collide`，就顺便再弄一个 `clamp` 函数

下面两段逻辑实际上就是限定 `self->image.x` 的取值范围 `[0, width - self->image.image->w]`，对于这样的需求，`clamp` 函数就很适合

``` c
// move left
if (self->image.x <= 0) {
    self->image.x = 0;
}
// move right
if (self->image.x + self->image.image->w >= width) {
    self->image.x = width - self->image.image->w;
}
```

定义工具函数 `clamp` 限定范围，再弄一个统一函数 `Paddle_move`，不管是往左边移动还是往右边移动，都调用 `Paddle_move`

``` c
int
clamp(int value, int min, int max) {
    if (value < min) {
        value = min;
    } else if (value > max) {
        value = max;
    }
    return value;
}

void
Paddle_move(Paddle *self, int x) {
    int x1 = clamp(x, 0, width - self->image.image->w);
    self->image.x = x1;
}

void
Paddle_moveLeft(Paddle *self) {
    Paddle_move(self, self->image.x - self->speed);
}

void
Paddle_moveRight(Paddle *self) {
    Paddle_move(self, self->image.x + self->speed);
}
```
