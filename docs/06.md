## 整理代码结构(2)
### 精简 main 函数

`main` 函数当前做的事情主要是
- 初始化 `SDL` 环境
- 创建 `Game`
- 创建 `Paddle`
- 运行一个大循环来处理核心操作
- 回收资源

先把大循环的代码抽出去，这样就可以精简 `main` 函数

``` c
void
Game_runLoop(Game *self) {
    Game *game = self;
    bool keyA_pressed = false;
    bool keyD_pressed = false;
    bool quit = false;
    while (quit == false) {
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_EVENT_QUIT) {
                quit = true;
            } else if (event.type == SDL_EVENT_KEY_DOWN) {
                if (event.key.key == SDLK_A) {
                    keyA_pressed = true;
                }
                if (event.key.key == SDLK_D) {
                    keyD_pressed = true;
                }
            } else if (event.type == SDL_EVENT_KEY_UP) {
                if (event.key.key == SDLK_A) {
                    keyA_pressed = false;
                }
                if (event.key.key == SDLK_D) {
                    keyD_pressed = false;
                }
            }
        }

        if (keyA_pressed) {
            paddle->moveLeft(paddle);
        }
        if (keyD_pressed) {
            paddle->moveRight(paddle);
        }

        Uint32 color = SDL_MapSurfaceRGB(game->surface, 255, 255, 255);
        SDL_FillSurfaceRect(game->surface, NULL, color);
        SDL_Rect rect = {
            .x = paddle->x,
            .y = paddle->y,
            .w = paddle->image->w,
            .h = paddle->image->h,
        };
        SDL_BlitSurface(paddle->image, NULL, game->surface, &rect);
        SDL_UpdateWindowSurface(game->window);
    }
}
```

抽出去之后会发现 `Game_runLoop` 里面用到了无法访问的 `paddle` 对象，但 `paddle` 是在 `main` 函数里面创建的

为了让 `Game_runLoop` 里面也能访问 `paddle`，有这么几个解决方案

1. 直接把 `paddle` 变成全局变量
2. 把 `paddle` 变成 `Game_runLoop` 的第二个参数
3. 让 `game` 持有 `paddle`

这里先采用方案三，把 `paddle` 变成 `game` 的成员

先在 `Game` 结构体里添加 `paddle` 变量
``` c
struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    Paddle *paddle;
};
typedef struct _Game Game;
```

创建 `game` 的时候，先把 `paddle` 初始化为 `NULL`
``` c
g->paddle = NULL;
```

然后在 `main` 函数里面创建 `paddle` 之后，赋值给 `game->paddle`，这里图方便创建 `paddle` 之后没有判断是否为空

``` c
Paddle *paddle = Paddle_new();
game->paddle = paddle;
Game_runLoop(game);
```

接着在 `Game_runLoop` 函数里面就可以通过 `game` 拿到 `paddle` 了

``` c
Game *game = self;
Paddle *paddle = self->paddle;
```

这样处理完之后，`main` 函数的逻辑就比较简单

``` c
int
main(int argc, char* argv[]) {
    if (SDL_Init(SDL_INIT_VIDEO) == false) {
        SDL_Log("Error initializing SDL: %s\n", SDL_GetError());
        return 1;
    }

    Game *game = Game_new();
    if (game == NULL) {
        SDL_Quit();
        return 1;
    }

    Paddle *paddle = Paddle_new();
    game->paddle = paddle;
    Game_runLoop(game);

    SDL_DestroySurface(paddle->image);
    free(paddle);
    SDL_DestroyWindow(game->window);
    free(game);
    SDL_Quit();

    return 0;
}
```

### 注册事件

`main` 函数的代码确实变少了，但 `Game_runLoop` 的逻辑并没有简化，依然要判断特定变量，然后根据变量值执行相关逻辑，这样做当然没有问题，不然程序也跑不起来

但其实这里想表达的逻辑是按了 `a` 往左边移动，按了 `d` 挡板往右边移动，换言之
- `a` 被按下这个事件，对应挡板往左移动
- `b` 被按下这个事件，对应挡板往右移动

看起来是一种映射关系，原则上这种映射关系可以换成哈希表来实现，但可惜 C 语言并没有内置哈希表这个数据结构，这种时候可以考虑用数组来模拟实现

数组 `keydowns` 存储按键的状态

数组 `actions` 存储当按键按下时执行的回调函数

先给 `Game` 添加这两个成员
``` c
#define MAX_COUNT 128

struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    Paddle *paddle;
    bool keydowns[MAX_COUNT];
    void (*actions[MAX_COUNT])(void);
};
```

结构体中的数组长度需要弄成编译时常量，说人话就是不支持变量，所以用宏来表示

接着在 `Game_new` 函数里对这两个成员进行初始化

常见方案是用 `memset` 来初始化，但是用 `for` 循环初始化也可以

``` c
for (int i = 0; i < MAX_COUNT; i += 1) {
    g->keydowns[i] = false;
    g->actions[i] = NULL;
}
```
