## 整理代码结构（二）
### 精简 main 函数

`main` 函数当前做的事情主要是
- 初始化 `SDL` 环境
- 创建 `Game`
- 创建 `Paddle`
- 运行一个大循环来处理核心操作
- 回收资源

先把大循环的代码抽出去，这样就可以精简 `main` 函数

``` c
void
Game_runLoop(Game *self) {
    Game *game = self;
    bool keyA_pressed = false;
    bool keyD_pressed = false;
    bool quit = false;
    while (quit == false) {
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_EVENT_QUIT) {
                quit = true;
            } else if (event.type == SDL_EVENT_KEY_DOWN) {
                if (event.key.key == SDLK_A) {
                    keyA_pressed = true;
                }
                if (event.key.key == SDLK_D) {
                    keyD_pressed = true;
                }
            } else if (event.type == SDL_EVENT_KEY_UP) {
                if (event.key.key == SDLK_A) {
                    keyA_pressed = false;
                }
                if (event.key.key == SDLK_D) {
                    keyD_pressed = false;
                }
            }
        }

        if (keyA_pressed) {
            paddle->moveLeft(paddle);
        }
        if (keyD_pressed) {
            paddle->moveRight(paddle);
        }

        Uint32 color = SDL_MapSurfaceRGB(game->surface, 255, 255, 255);
        SDL_FillSurfaceRect(game->surface, NULL, color);
        SDL_Rect rect = {
            .x = paddle->x,
            .y = paddle->y,
            .w = paddle->image->w,
            .h = paddle->image->h,
        };
        SDL_BlitSurface(paddle->image, NULL, game->surface, &rect);
        SDL_UpdateWindowSurface(game->window);
    }
}
```

抽出去之后会发现 `Game_runLoop` 里面用到了无法访问的 `paddle` 对象，但 `paddle` 是在 `main` 函数里面创建的

为了让 `Game_runLoop` 里面也能访问 `paddle`，有这么几个解决方案

1. 直接把 `paddle` 变成全局变量
2. 把 `paddle` 变成 `Game_runLoop` 的第二个参数
3. 让 `game` 持有 `paddle`

这里先采用方案三，把 `paddle` 变成 `game` 的成员

先在 `Game` 结构体里添加 `paddle` 变量
``` c
struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    Paddle *paddle;
};
typedef struct _Game Game;
```

创建 `game` 的时候，先把 `paddle` 初始化为 `NULL`
``` c
g->paddle = NULL;
```

然后在 `main` 函数里面创建 `paddle` 之后，赋值给 `game->paddle`，这里图方便创建 `paddle` 之后没有判断是否为空

``` c
Paddle *paddle = Paddle_new();
game->paddle = paddle;
Game_runLoop(game);
```

接着在 `Game_runLoop` 函数里面就可以通过 `game` 拿到 `paddle` 了

``` c
Game *game = self;
Paddle *paddle = self->paddle;
```

这样处理完之后，`main` 函数的逻辑就比较简单

``` c
int
main(int argc, char* argv[]) {
    if (SDL_Init(SDL_INIT_VIDEO) == false) {
        SDL_Log("Error initializing SDL: %s\n", SDL_GetError());
        return 1;
    }

    Game *game = Game_new();
    if (game == NULL) {
        SDL_Quit();
        return 1;
    }

    Paddle *paddle = Paddle_new();
    game->paddle = paddle;
    Game_runLoop(game);

    SDL_DestroySurface(paddle->image);
    free(paddle);
    SDL_DestroyWindow(game->window);
    free(game);
    SDL_Quit();

    return 0;
}
```

### 注册事件

`main` 函数的代码确实变少了，但 `Game_runLoop` 的逻辑并没有简化，依然要判断特定变量，然后根据变量值执行相关逻辑，这样做当然没有问题，不然程序也跑不起来

但其实这里想表达的逻辑是按了 `a` 往左边移动，按了 `d` 挡板往右边移动，换言之
- `a` 被按下这个事件，对应挡板往左移动
- `b` 被按下这个事件，对应挡板往右移动

看起来是一种映射关系，原则上这种映射关系可以换成哈希表来实现，但可惜 C 语言并没有内置哈希表这个数据结构，这种时候可以考虑用数组来模拟实现

数组 `keydowns` 存储按键的状态

数组 `actions` 存储当按键按下时执行的回调函数

先给 `Game` 添加这两个成员
``` c
#define MAX_COUNT SDL_SCANCODE_COUNT

struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    Paddle *paddle;
    bool keydowns[MAX_COUNT];
    void (*actions[MAX_COUNT])(void);
};
```

结构体中的数组长度需要弄成编译时常量，说人话就是不支持变量，所以用宏来表示

接着在 `Game_new` 函数里对这两个成员进行初始化

常见方案是用 `memset` 来初始化，但是用 `for` 循环初始化也可以

``` c
for (int i = 0; i < MAX_COUNT; i += 1) {
    g->keydowns[i] = false;
    g->actions[i] = NULL;
}
```

接下来实现一个注册按键的功能，这样就可以通过按键与对应的回调函数映射在一起

``` c
void
Game_registerAction(Game *self, SDL_Scancode key, void (*callback)(void)) {
    assert(key < MAX_COUNT);
    Game *game = self;
    game->actions[key] = callback;
}
```

用法也很简单，依次传入 `game` 对象，按键与函数，当然函数需要提前定义好，`SDL_SCANCODE_A` 是 `SDL` 提供的用来表示 `a` 的键

``` c
void
moveLeft(void) {
    SDL_Log("register key and move left\n");
}

Game_registerAction(game, SDL_SCANCODE_A, moveLeft);
```

然后在主循环里遍历整个 `keydowns` 数组与 `actions` 数组，如果 `keydowns` 里面有值并且对应下标的 `actions` 也有值，说明注册了按键，就执行对应的回调函数

当然在遍历之前，还需要把按键存入 `keydowns` 数组中，按下键 `keydowns` 数组存 `true`，抬起键 `keydowns` 数组存 `false`

这样就可以去掉用不上的相关变量了

为了和前面 `Game_registerAction` 对应，把 `event.key.key` 也同步改成了 `event.key.scancode`，因为 `event.key.key` 定义的范围更大，如果要用数组来存储，那么这个数组会非常大，而 `event.key.scancode` 的范围小很多，更适合

``` c
void
Game_runLoop(Game *self) {
    Game *game = self;
    Paddle *paddle = self->paddle;

    Game_registerAction(game, SDL_SCANCODE_A, moveLeft);

    bool quit = false;
    while (quit == false) {
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_EVENT_QUIT) {
                quit = true;
            } else if (event.type == SDL_EVENT_KEY_DOWN) {
                SDL_Scancode sc = event.key.scancode;
                if (sc < MAX_COUNT) {
                    game->keydowns[sc] = true;
                }
            } else if (event.type == SDL_EVENT_KEY_UP) {
                SDL_Scancode sc = event.key.scancode;
                if (sc < MAX_COUNT) {
                    game->keydowns[sc] = false;
                }
            }
        }

        for (int i = 0; i < MAX_COUNT; i += 1) {
            if (game->actions[i] != NULL && game->keydowns[i]) {
                game->actions[i]();
            }
        }

        Uint32 color = SDL_MapSurfaceRGB(game->surface, 255, 255, 255);
        SDL_FillSurfaceRect(game->surface, NULL, color);
        SDL_Rect rect = {
            .x = paddle->x,
            .y = paddle->y,
            .w = paddle->image->w,
            .h = paddle->image->h,
        };
        SDL_BlitSurface(paddle->image, NULL, game->surface, &rect);
        SDL_UpdateWindowSurface(game->window);
    }
}
```

### 自定义参数
程序改到这一步已经非常精简，但依然有一个问题，现在按 `a` 键之后，`paddle` 并不会往左边移动，因为 `moveLeft` 里面没办法获取 `paddle` 的值

为了解决这个问题，需要改造 `Game_registerAction`，增加一个参数专门用来传值，而且可以想象得到，不同按键对应的参数可能也有差异，所以仿照 `keydowns` 弄一个参数数组

需要注意，这里并不能修改第二个参数，假设把 `void (*callback)(void)` 改成 `void (*callback)(Paddle *paddle)` 那也只适用于 `paddle` 的情况，按键之后如果需要处理其他对象就做不到

``` c
struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    Paddle *paddle;
    bool keydowns[MAX_COUNT];
    void (*actions[MAX_COUNT])(void *);
    void *data[MAX_COUNT];
};
```

`data` 是长度为 `MAX_COUNT` 的数组，`void *` 表示每一个元素都是指向 `void` 的指针，在 C 语言里面，`void *` 可以转换为其他类型

同时需要注意，`actions` 原本存的函数没有参数，现在也要补一个 `void *` 类型的参数

在结构体里新增成员之后，同样需要初始化

``` c
for (int i = 0; i < MAX_COUNT; i += 1) {
    g->keydowns[i] = false;
    g->actions[i] = NULL;
    g->data[i] = NULL;
}
```

`Game_registerAction` 第三个参数的类型也要同步修改，并且还增加了第四个参数，同时把第四个参数保存到 `data` 数组中

``` c
void
Game_registerAction(Game *self, SDL_Scancode key, void (*callback)(void *), void *data) {
    assert(key < MAX_COUNT);
    Game *game = self;
    game->actions[key] = callback;
    game->data[key] = data;
}
```

调用的时候传入 `paddle` 作为第四个参数，接着在回调函数也就是 `moveLeft` 定义的地方补充参数，在 `moveLeft` 内部把 `void *` 转成需要的 `Paddle *`，这样就可以调用 `Paddle` 的方法了

``` c
void
moveLeft(void *data) {
    Paddle *paddle = (Paddle *)data;
    Paddle_moveLeft(paddle);
}

Game_registerAction(game, SDL_SCANCODE_A, moveLeft, paddle);
```

循环里调用 `actions` 回调的时候同步传入参数
``` c
for (int i = 0; i < MAX_COUNT; i += 1) {
    if (game->actions[i] != NULL && game->keydowns[i]) {
        game->actions[i](game->data[i]);
    }
}
```

### 帧率问题
把主要逻辑放在无限循环里来处理有一个潜在的问题，不同性能的电脑表现出来会有差异，理论上 CPU 越强，循环执行次数越多，最终可能表现出来不是 60fps，而是 120fps 甚至更多

通常会希望稳定在 60fps，也就是一次循环执行 16.6ms，如果一次循环耗费非常快，低于 16.6ms，那就需要刻意等到 16.6ms

所以需要在循环开始的时候用 `SDL_GetTicks` 记录初始时间，在渲染结束再次用 `SDL_GetTicks` 记录结束时间，两次时间低于 16.6ms 的时候，用 `SDL_Delay` 等时间过去
``` c
const int delay = 1000 / 60;
while (true) {
    Uint32 frameStart = SDL_GetTicks();
    // 主循环处理各种任务
    Uint64 frameEnd = SDL_GetTicks();
    int frameTime = frameEnd - frameStart;
    if (frameTime < delay) {
        SDL_Delay(delay - frameTime);
    }
}
```
