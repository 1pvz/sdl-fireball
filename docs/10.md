## debug 工具
### 暂停功能
程序很难一次性写对，所以方便的调试方案很重要

对于游戏来说，始终会运行一个游戏循环，这点与 Web 程序很不一样。假设 `ball` 与 `block` 碰撞后，`block` 没有消失（如果碰撞函数实现不对，这很容易发生），那调试就会比较麻烦，最好是希望能 “抓住” 那一帧的状态

这意味着程序要有暂停功能，暂停之后，场景里的元素都不运动，这样就方便调试，因为目前没有音效和背景音乐的需求，所以可以把整个游戏直接暂停

停掉游戏的逻辑很简单，现在所有逻辑都放在 `Game_update` 里完成，所以直接中断这个函数的执行

先给 `game` 增加属性 `paused`，创建的时候初始化为 `false`

``` c
// game.h
struct _Game {
    // 其他属性
    bool paused;
};
```

`Game_update` 里面判断 `paused` 如果为 `true` 就不让函数继续执行

``` c
// game.c
void
Game_update(Game *self) {
    Game *game = self;
    if (game->paused) {
        return;
    }
    Ball_move(game->ball);
    if (Paddle_collide(game->paddle, game->ball->image)) {
        Ball_bounce(game->ball);
    }
    for (int i = 0; i < game->numberOfBlocks; i += 1) {
        Block *block = game->blocks[i];
        if (Block_collide(block, game->ball->image)) {
            Block_kill(block);
            Ball_bounce(game->ball);
        }
    }
}
```

接下来就是处理如何改变 `paused` 的值，最方便的方案是直接绑定按键，比如说按了 `p` 键就暂停，再次按 `p` 就恢复游戏，也就是每次按下 `p` 都会对 `paused` 进行取反操作

在 `SDL_EVENT_KEY_DOWN` 事件中补充这个判断

``` c
if (event.type == SDL_EVENT_KEY_DOWN) {
    SDL_Scancode sc = event.key.scancode;
    if (sc < MAX_COUNT) {
        game->keydowns[sc] = true;
    }
    if (sc == SDL_SCANCODE_P) {
        game->paused = !game->paused;
    }
}
```

### 拖拽

暂停是为了方便拿到状态，最好还能直接复现状态，比如让 `ball` 撞击 `block`，但 `ball` 的运行路线我们很难控制，自然就不容易出现预期的碰撞行为

最好是手动把 `ball` 放在特定位置，比如就放在 `block` 旁边，那就很容易发生碰撞了

虽然可以在代码里面直接指定 `ball` 的位置，但需要反复尝试比较麻烦，所以想直接拖动小球到特定为止

拖动的逻辑是按住鼠标左键不松开，拖到哪里 `ball` 就跟到哪里

在 `game` 里面增加 `mouseDown` 用来记录是否按下鼠标，初始值为 `false`

``` c
// game.h
struct _Game {
    // 其他属性
    bool mouseDown;
};
```

监听多个事件
- `SDL_EVENT_MOUSE_BUTTON_DOWN` 鼠标按下的时候，`mouseDown` 设置为 `true`
- `SDL_EVENT_MOUSE_BUTTON_UP` 鼠标抬起的时候，`mouseDown` 设置为 `false`
- `SDL_EVENT_MOUSE_MOTION` 鼠标移动的时候，如何 `mouseDown` 为 `true`，说明此时鼠标依然保持按下的状态，也就是按住并且移动，这个时候判断如果鼠标选中了 `ball` 就更新 `ball` 的坐标

``` c
if (event.type == SDL_EVENT_MOUSE_BUTTON_DOWN) {
    game->mouseDown = true;
} else if (event.type == SDL_EVENT_MOUSE_BUTTON_UP) {
    game->mouseDown = false;
} else if (event.type == SDL_EVENT_MOUSE_MOTION) {
    if (game->mouseDown) {
        float x = event.motion.x;
        float y = event.motion.y;
        SDL_Point point = {
            .x = x,
            .y = y,
        };
        SDL_Rect rect = {
            .x = game->ball->image.x,
            .y = game->ball->image.y,
            .w = game->ball->image.image->w,
            .h = game->ball->image.image->h,
        };
        if (SDL_PointInRect(&point, &rect)) {
            SDL_Log("x and y is %f %f\n", x, y);
            game->ball->image.x = x;
            game->ball->image.y = y;
        }
    }
}
```

但这种方案有一个缺陷，在拖动小球的时候会觉得 "不跟手"，而且基本上没可能往左上方向移动

想象下 `ball` 的大小为 20 * 20，并且渲染在 `(80, 80)` 的位置

假设鼠标点击在 `ball` 10 * 10 的位置，恰好是 `ball` 中心点，对应坐标为 `(90, 90)`

此时鼠标往右边移动到 `(95, 90)`，那么 `ball` 左上角也会移动到 `(95, 90)`，结果表现出来就是鼠标刚好处于 `ball` 的左上角顶点，而不是原本中心的位置

如果鼠标往左上方移动到 `(92, 87)`，就不会与 `ball` 重合（`ball` 的四个点坐标分别为 `(95, 90) (115, 90) (115, 110) (95, 110)`)，自然就没办法拖动

所以还需要记录鼠标与 `ball` 重合的时候，鼠标与 `ball` 左上角的偏移量

然后思路转变下，在鼠标按下的时候就记录能不能拖动，然后在鼠标拖动的时候再判断需要拖到什么位置，这样用 `mouseDown` 就不合适了，用 `enableDrag` 来表示，并且删除 `mousedown`

``` c
// game.h
struct _Game {
    // 其他属性
    bool enableDrag;
    float offsetX;
    float offsetY;
};
```

``` c
if (event.type == SDL_EVENT_MOUSE_BUTTON_DOWN) {
    float x = event.button.x;
    float y = event.button.y;
    SDL_Point point = {
        .x = x,
        .y = y,
    };
    SDL_Rect rect = {
        .x = game->ball->image.x,
        .y = game->ball->image.y,
        .w = game->ball->image.image->w,
        .h = game->ball->image.image->h,
    };
    if (SDL_PointInRect(&point, &rect)) {
        game->enableDrag = true;
        game->offsetX = x - rect.x;
        game->offsetY = y - rect.y;
    }
} else if (event.type == SDL_EVENT_MOUSE_BUTTON_UP) {
    game->enableDrag = false;
} else if (event.type == SDL_EVENT_MOUSE_MOTION) {
    if (game->enableDrag) {
        float x = event.motion.x;
        float y = event.motion.y;
        game->ball->image.x = x - game->offsetX;
        game->ball->image.y = y - game->offsetY;
    }
}
```

### 配置 fps
除了增加暂停与拖拽功能，还需要一个调整 `fps` 的功能，调试的时候，`fps` 值较小会更容易观察结果

把这个属性放在 `config` 里面

``` c
struct _GameConfig {
    int width;
    int height;
    int fps;
};

struct _Game {
    // 其他属性
    int fps;
};
```

创建 `game` 的时候传过去，并且在 `Game_new` 里面赋值

``` c
Game *game = Game_new((GameConfig) {
    .width = 400,
    .height = 300,
    .fps = 10,
});

g->fps = gameConfig.fps;
```

然后 `Game_runloop` 里面更改就可以
``` c
const int delay = 1000 / game->fps;
```

### 加载关卡
按照现在的逻辑，其实完全可以把 `block` 的初始位置提前写好，三个位置分别为 `(0, 100)` `(100, 100)` `(200, 100)`，对于这样的数据，直接保存在数组中就可以

``` c
int positions[NUMBER_OF_BLOCKS][2] = {
    {0, 100},
    {100, 100},
    {200, 100},
};
for (int i = 0; i < NUMBER_OF_BLOCKS; i += 1) {
    // 其他代码
    block->image.x = positions[i][0];
    block->image.y = positions[i][1];
    // 其他代码
}
```

但可以想象，关卡的数据会越来越多，如果都放在 `main` 函数里面直接写会不方便，所以会想抽到单独的配置文件里，大概会写成下面形式的代码

在其他语言的确可行，但是 C 语言并不支持返回完整数组

``` c
数组
loadPositions(void) {
    int positions[3][2] = {
        {0, 100},
        {100, 100},
        {200, 100},
    };
    return positions;
}
```

为了简化问题，先从一维数组开始考虑，既然不能直接返回完整数组，那返回指针好像可行

`int` 一维数组就返回 `int *`，表示指向这个一维数组第一个元素的地址

``` c
int *
loadPosition(void) {
    int position[2] = {0, 100};
    return position;
}
```

但这么做依然不行，因为 `position` 数组直接分配在栈上的内存，`loadPosition` 函数结束之后，这个内存会被回收

既然数组行不通，就换一个思路，把位置信息保存在结构体里，然后返回结构体，就可以拿到位置信息

```c
struct _Position {
    int x;
    int y;
};
typedef struct _Position Position;

Position
loadPosition(void) {
    Position p1 = {
        .x = 0,
        .y = 100,
    };
    return p1;
}
```

接下来需要处理 3 个位置信息，因为直接分配在栈上的内存会在函数结束之后回收，所以需要处理成指针的形式，当然这里也可以继续包一层包含数组的结构体，然后返回整个结构体

``` c
Position *
loadPosition(void) {
    Position p1 = {
        .x = 0,
        .y = 100,
    };
    Position p2 = {
        .x = 100,
        .y = 100,
    };
    Position p3 = {
        .x = 200,
        .y = 100,
    };
    Position *array = malloc(sizeof(Position) * 3);
    if (array == NULL) {
        return NULL;
    }
    array[0] = p1;
    array[1] = p2;
    array[2] = p3;
    return array;
}
```

可以发现写成这样其实很麻烦，理想情况还是用字面量的形式来声明数组，但数组又不能直接返回

通常遇到这种情况，可以考虑直接处理成“全局”变量的形式

在 c 文件里面定义的变量，外界不能直接访问，所以很适合直接声明关卡数据

``` c
Position level1[] = {
    (Position){
        .x = 0,
        .y = 100,
    },
    (Position){
        .x = 100,
        .y = 100,
    },
    (Position){
        .x = 200,
        .y = 100,
    },
};
```

`level1` 数组的元素都是 `Position` 类型，有 3 个元素，但问题是代码并不知道有 3 个元素，所以还需要在现有结构的基础上额外加一个字段表示数量

``` c
struct _LevelConfig {
    int numberOfBlocks;
    Position *positions;
};
typedef struct _LevelConfig LevelConfig;

LevelConfig levelConfig = {
    .numberOfBlocks = 3,
    .positions = level1,
};

LevelConfig
loadLevelConfig(void) {
    return levelConfig;
}
```

在 `main` 函数里面调用，可以拿到对应的坐标值

``` c
LevelConfig levelConfig = loadLevelConfig();
Position *position = levelConfig.positions;
for (int i = 0; i < levelConfig.numberOfBlocks; i += 1) {
    Position p = position[i];
    SDL_Log("position is (%d, %d)\n", p.x, p.y);
}
```

可以拿到坐标之后，就要考虑替换掉现有的方案，即直接在 `main` 函数里面定义 `positions` 的方案

原本在 `game` 里定义了固定长度的数组 `Block *blocks[NUMBER_OF_BLOCKS]`，数组的长度与坐标长度对应，但现在应该换成不定长度的数组，因为并不知道一个关卡有多少个 `block`

不定长数组最简单的方案是 `Block **blocks`，也就是指向 `Block *` 的指针，因为暂时并不知道一共有多少个 `block`，所以初始化的时候值设为 `NULL`

``` c
// for (int i = 0; i < NUMBER_OF_BLOCKS; i += 1) {
//     g->blocks[i] = NULL;
// }
g->blocks = NULL;
```

在 `main` 函数里面调用 `loadLevelConfig` 拿到关卡配置后就可以知道有多少个 `block`，循环遍历拿到 `position` 的值，然后创建对应的 `block`，最后再统一赋值给 `game`，这样就可以用动态数组来维护 `block`

``` c
LevelConfig levelConfig = loadLevelConfig();
Block **blocks = malloc(levelConfig.numberOfBlocks * sizeof(Block *));
if (blocks == NULL) {
    SDL_Log("Failed to create blocks\n");
    SDL_DestroyWindow(game->window);
    free(game);
    SDL_Quit();
    return 1;
}
Position *position = levelConfig.positions;
for (int i = 0; i < levelConfig.numberOfBlocks; i += 1) {
    Block *block = Block_new_with_image(blockImage);
    if (block == NULL) {
        // 清理已经创建成功的 block
        for (int j = 0; j < i; j += 1) {
            free(game->blocks[j]);
        }
        SDL_Log("Failed to create block\n");
        SDL_DestroyWindow(game->window);
        free(game);
        SDL_Quit();
        return 1;
    }
    block->image.x = position[i].x;
    block->image.y = position[i].y;
    blocks[i] = block;
    game->numberOfBlocks += 1;
}
game->blocks = blocks;
```
