## 回调函数
### 按钮
`SDL` 原生没有提供按钮功能，所以需要自己实现按钮，考虑最朴素的情况，按钮是一个矩形，连文本都可以去掉，在 `main` 函数外面定义一个这样的结构体

``` c
struct _Button {
    SDL_Rect rect;
};
typedef struct _Button Button;
```

有了 `Button` 结构体之后，定义一个按钮，并且在 `surface` 上面画出来

``` c
SDL_Rect rect = {
    .x = 100,
    .y = 100,
    .w = 120,
    .h = 50,
};

Button button = {
    .rect = rect,
};

bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        }
    }

    Uint32 color = SDL_MapSurfaceRGB(surface, 255, 255, 255);
    SDL_FillSurfaceRect(surface, NULL, color);

    Uint32 buttonColor = SDL_MapSurfaceRGB(surface, 255, 0, 0);
    SDL_FillSurfaceRect(surface, &button.rect, buttonColor);
    SDL_UpdateWindowSurface(window);
}
```

运行之后会在 `(100, 100)` 的位置画一个 `120 * 50` 的红色实心矩形

接下来希望给按钮增加几个状态，比如鼠标是否 hover 上去，以及鼠标是否点击了按钮

``` c
struct _Button {
    SDL_Rect rect;
    bool is_hover;
    bool is_pressed;
};
typedef struct _Button Button;
```

相应地，创建 button 的时候，也要初始化这两个值
``` c
Button button = {
    .rect = rect,
    .is_hover = false,
    .is_pressed = false,
};
```

移动鼠标的时候，可以获取鼠标指针的坐标点，假设这个点在 button 矩形内部，就认为按钮处于 hover 状态

鼠标移动的事件是 `SDL_EVENT_MOUSE_MOTION`，鼠标移动后，通过 `event.motion` 获取鼠标的坐标，根据坐标信息更新 hover 状态

并且在设置按钮颜色的时候，根据 hover 状态设置不同颜色

最后表现为默认情况按钮为红色，鼠标 hover 上去按钮为绿色

``` c
bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        } else if (event.type == SDL_EVENT_MOUSE_MOTION) {
            int mouseX = event.motion.x;
            int mouseY = event.motion.y;
            SDL_Log("mouse motion %d and %d\n", mouseX, mouseY);
            bool xIn = rect.x <= mouseX && mouseX <= rect.x + rect.w;
            bool yIn = rect.y <= mouseY && mouseY <= rect.y + rect.h;
            button.is_hover = xIn && yIn;
        }
    }

    Uint32 color = SDL_MapSurfaceRGB(surface, 255, 255, 255);
    SDL_FillSurfaceRect(surface, NULL, color);

    Uint32 buttonColor = SDL_MapSurfaceRGB(surface, 255, 0, 0);
    if (button.is_hover) {
        buttonColor = SDL_MapSurfaceRGB(surface, 0, 255, 0);
    }
    SDL_FillSurfaceRect(surface, &button.rect, buttonColor);
    SDL_UpdateWindowSurface(window);
}
```

判断点是否在矩形内是常见需求，`SDL` 提供了 `SDL_PointInRect` 函数来简化操作

``` c
int mouseX = event.motion.x;
int mouseY = event.motion.y;
SDL_Point point = {
    .x = mouseX,
    .y = mouseY,
};
SDL_Log("mouse motion %d and %d\n", mouseX, mouseY);
button.is_hover = SDL_PointInRect(&point, &button.rect);
```

考虑一种情况，假设检测到鼠标按下了，并且此时鼠标在按钮内（也就是按钮处于 hover 状态），就可以认为鼠标点击了按钮

``` c
if (event.type == SDL_EVENT_QUIT) {
    quit = true;
} else if (event.type == SDL_EVENT_MOUSE_MOTION) {
    int mouseX = event.motion.x;
    int mouseY = event.motion.y;
    SDL_Point point = {
        .x = mouseX,
        .y = mouseY,
    };
    SDL_Log("mouse motion %d and %d\n", mouseX, mouseY);
    button.is_hover = SDL_PointInRect(&point, &button.rect);
} else if (event.type == SDL_EVENT_MOUSE_BUTTON_DOWN) {
    if (button.is_hover) {
        button.is_pressed = true;
    }
} else if (event.type == SDL_EVENT_MOUSE_BUTTON_UP) {
    button.is_pressed = false;
}
```

现在没有给按钮绑定回调事件，就先通过颜色来体现这点

需要注意，因为现在 `is_pressed` 为 true 的时候，`is_hover` 必定为 true，所以这两个状态有一个优先级

为了方便观察，把 `is_pressed` 的判断放在更前面

- 按钮按下，按钮变成蓝色
- 按钮没按下但处于 hover 状态，按钮变成绿色
- 按钮没处于 hover 状态，按钮变成红色

``` c
Uint32 buttonColor;
if (button.is_pressed) {
    buttonColor = SDL_MapSurfaceRGB(surface, 0, 0, 255);
} else if (button.is_hover) {
    buttonColor = SDL_MapSurfaceRGB(surface, 0, 255, 0);
} else {
    buttonColor = SDL_MapSurfaceRGB(surface, 255, 0, 0);
}
```

### 事件回调函数
给按钮加上相关状态后，就可以给按钮绑定事件回调函数，比如给 click 事件绑定回调函数，当按钮被点击的时候会触发对应的回调函数

C 语言没办法像 JavaScript 那样直接调用 `addEventListener` 然后传入匿名函数作为回调函数的参数

C 语言需要用函数指针的形式来处理，`void (*on_button_click)(struct _Button *button)` 就是函数指针，遇到函数指针需要从中间往外面读
- `on_button_click` 变量名称
- `*on_button_click` 表示这是一个指针变量
- `(*on_button_click)(struct _Button *button)` 表示这个指针指向一个函数，函数有一个参数，参数类型是 `struct _Button *`，参数名称是 `button`，当然也可以省略参数名称写成 `void (*on_button_click)(struct _Button *)` 的形式
- `void` 表示函数没有返回值

综合起来，`void (*on_button_click)(struct _Button *button)` 指 `on_button_click` 是一个函数指针，指向参数类型为 `struct _Button *` 且返回值为 `void` 的函数

``` c
struct _Button {
    SDL_Rect rect;
    bool is_hover;
    bool is_pressed;
    void (*on_button_click)(struct _Button *button);
};
```

接着定义回调函数，回调函数里面会把 button 的两个状态 log 出来

``` c
void
onButtonClick(Button *button) {
    SDL_Log("button is_hover = %d, is_pressed = %d\n", button->is_hover, button->is_pressed);
}
```

接着在初始化 `button` 的时候，补充 `on_button_click` 的值

``` c
Button button = {
    .rect = rect,
    .is_hover = false,
    .is_pressed = false,
    .on_button_click = onButtonClick,
};
```

然后就是在点击的时候判断回调函数，如果不为空则认为有回调函数，此时调用回调函数并且把 button 作为参数传过去

``` c
if (button.is_hover) {
    button.is_pressed = true;
    if (button.on_button_click != NULL) {
        button.on_button_click(&button);
    }
}
```
