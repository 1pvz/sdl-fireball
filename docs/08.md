## 添加 block
有 `paddle` 和 `ball` 之后，还需要补充 `block`，`block` 不会移动，被击中之后会消失

``` c
struct _Block {
    GuaImage image;
    bool alive;
};
typedef struct _Block Block;

Block *
Block_new(void) {
    SDL_Surface *image = imageFromPath("block.png");
    if (image == NULL) {
        return NULL;
    }
    Block *o = malloc(sizeof(Block));
    if (o == NULL) {
        SDL_DestroySurface(image);
        return NULL;
    }
    o->image.image = image;
    o->image.x = 100;
    o->image.y = 100;
    o->alive = true;
    return o;
}

void
Block_kill(Block *self) {
    self->alive = false;
}
```

`main` 函数里面创建 `block`，并且最后释放资源

``` c
Block *block = Block_new();
if (block == NULL) {
    SDL_Log("Failed to create block\n");
    SDL_DestroyWindow(game->window);
    free(game);
    SDL_Quit();
    return 1;
}
game->block = block;
// ...
// 游戏主循环
// ...
SDL_DestroySurface(block->image.image);
free(block);
```

`Game` 自然需要拿到 `Block`，也就是 `Game` 的 `struct` 里面需要补充 `block` 属性，初始化的时候设置为 `NULL`，`main` 函数里面创建 `block` 之后就挂在 `game` 上

这样才可以在 `Game_draw` 函数里面画出 `block`

``` c
if (block->alive) {
    Game_drawImage(game, block->image);
}
```

`Game_update` 里面还需要判断碰撞，碰撞之后调用 `Block_kill` 把 `block` 移除，同时让 `ball` 反弹
``` c
void
Game_update(Game *self) {
    Game *game = self;
    Ball_move(game->ball);
    if (collide(game->paddle->image, game->ball->image)) {
        Ball_bounce(game->ball);
    }
    if (collide(game->block->image, game->ball->image)) {
        Block_kill(game->block);
        Ball_bounce(game->ball);
    }
}
```

`collide` 目前是通用函数，也可以考虑放在对象上，比如分别让 `paddle` 和 `block` 都有 `collide` 函数
``` c
bool
Paddle_collide(Paddle *self, GuaImage image) {
    return collide(self->image, image);
}

bool
Block_collide(Block *self, GuaImage image) {
    return collide(self->image, image);
}

void
Game_update(Game *self) {
    Game *game = self;
    Ball_move(game->ball);
    if (Paddle_collide(game->paddle, game->ball->image)) {
        Ball_bounce(game->ball);
    }
    if (Block_collide(game->block, game->ball->image)) {
        Block_kill(game->block);
    }
}
```

不过这样操作可能会出现一个问题，因为是在 `Paddle_collide` 里面调用 `collide` 函数，C 语言要求函数先定义才能再使用，所以如果 `collide` 定义在 `Paddle_collide` 后面就会报错

当然可以让 `collide` 放在最前面定义，但是只要程序规模稍微增加一点点，这种通过人工定义顺序的方式就比较不方便，更好的办法是先声明 `collide` 函数

`collide` 需要用到 `GuaImage` 类型，所以在 `GuaImage` 后面声明，声明相当于告诉编译器有这么个函数，这样编译器看到调用 `collide` 的时候就不会报错

至于 `collide` 函数的具体实现不用管

``` c
struct _GuaImage {
    SDL_Surface *image;
    int x;
    int y;
};
typedef struct _GuaImage GuaImage;

bool
collide(GuaImage a, GuaImage b);
```

现在会有一个新问题，`ball` 与 `block` 相撞之后，如果 `ball` 再次移动到 `block` 消失之前的位置，还是会反弹，因为重新走了一遍 `Game_update` 的碰撞逻辑，所以碰撞的时候需要限定只有 `alive` 为 `true` 的 `block` 才能参与碰撞，把 `collide` 封装为函数之后，增加这个判断很方便

``` c
bool
Block_collide(Block *self, GuaImage image) {
    return self->alive && collide(self->image, image);
}
```

### 添加多个 block
一个 `block` 的情况跑通之后，就可以添加多个 `block`，比如添加 3 个 `block`，涉及到的修改并不多

`Game` 结构体里先添加成员

``` c
#define NUMBER_OF_BLOCKS 3

struct _Game {
    SDL_Window *window;
    SDL_Surface *surface;
    bool quit;
    Paddle *paddle;
    Ball *ball;
    Block *blocks[NUMBER_OF_BLOCKS];
    int numberOfBlocks;
    bool keydowns[MAX_COUNT];
    void (*actions[MAX_COUNT])(void *);
    void *data[MAX_COUNT];
};
```

创建 `game` 的时候初始化成员

``` c
for (int i = 0; i < NUMBER_OF_BLOCKS; i += 1) {
    g->blocks[i] = NULL;
}
g->numberOfBlocks = 0;
```

`main` 函数里面创建 3 个 `block`，程序结束之后销毁

``` c
for (int i = 0; i < NUMBER_OF_BLOCKS; i += 1) {
    Block *block = Block_new();
    if (block == NULL) {
        SDL_Log("Failed to create block\n");
        SDL_DestroyWindow(game->window);
        free(game);
        SDL_Quit();
        return 1;
    }
    block->image.x = i * 100;
    block->image.y = 100;
    game->blocks[i] = block;
    game->numberOfBlocks += 1;
}
// ...
// 游戏主循环
// ...
for (int i = 0; i < game->numberOfBlocks; i += 1) {
    Block *block = game->blocks[i];
    SDL_DestroySurface(block->image.image);
    free(block);
    game->blocks[i] = NULL;
}
game->numberOfBlocks = 0;
```

`update` 和 `draw` 里面的逻辑也换成遍历 `blocks` 数组

``` c
for (int i = 0; i < game->numberOfBlocks; i += 1) {
    Block *block = game->blocks[i];
    if (Block_collide(block, game->ball->image)) {
        Block_kill(block);
        Ball_bounce(game->ball);
    }
}
```

``` c
for (int i = 0; i < game->numberOfBlocks; i += 1) {
    Block *block = game->blocks[i];
    if (block->alive) {
        Game_drawImage(game, block->image);
    }
}
```

### 共享图片
每个 `block` 都会调用 `imageFromPath` 加载同一张图片，必要性不大，所以可以考虑先加载图片，然后所有 `block` 都用这张图片来初始化

``` c
Block *
Block_new_with_image(SDL_Surface *image) {
    Block *o = malloc(sizeof(Block));
    if (o == NULL) {
        // 多个 block 共享图片，清理工作放在最后
        // SDL_DestroySurface(image);
        return NULL;
    }
    o->image.image = image;
    o->image.x = 100;
    o->image.y = 100;
    o->alive = true;
    return o;
}
```

``` c
SDL_Surface *blockImage = imageFromPath("block.png");
if (blockImage == NULL) {
    SDL_Log("Failed to load block image\n");
    SDL_DestroyWindow(game->window);
    free(game);
    SDL_Quit();
    return 1;
}
for (int i = 0; i < NUMBER_OF_BLOCKS; i += 1) {
    Block *block = Block_new_with_image(blockImage);
    if (block == NULL) {
        // 清理已经创建成功的 block
        for (int j = 0; j < i; j += 1) {
            free(game->blocks[j]);
        }
        SDL_Log("Failed to create block\n");
        SDL_DestroyWindow(game->window);
        free(game);
        SDL_Quit();
        return 1;
    }
    block->image.x = i * 100;
    block->image.y = 100;
    game->blocks[i] = block;
    game->numberOfBlocks += 1;
}

for (int i = 0; i < game->numberOfBlocks; i += 1) {
    Block *block = game->blocks[i];
    free(block);
    game->blocks[i] = NULL;
}
game->numberOfBlocks = 0;
// 共享图片后，只需要清理一次
SDL_DestroySurface(blockImage);
```
