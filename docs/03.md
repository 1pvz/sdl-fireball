## 事件
`SDL` 使用事件来处理用户的输入，当用户使用鼠标或者键盘时，会产生对应的事件

`SDL` 会把该事件存到**事件队列**中，队列是一个先进先出的数据结构，即事件从队列尾部加入，从队列首部取出。

为了及时响应这些事件，会用一个 `event loop（事件循环）` 的方案来应对

每次运行事件循环，会从队列首部取出一个事件并做出相应操作，从队列首部取出事件后，队列的事件数量就会减少，如果队列里面没有事件，就成了一个空事件队列，循环也就不再继续。

前面的例子中都是用 `SDL_WaitEvent` 从事件队列中获取事件，但在游戏中希望能实时响应事件，用 `SDL_PollEvent` 更适合

程序运行一个无限循环，如果没有任何事件发生，事件队列为空，从队列首部无法获取事件，`SDL_PollEvent(&event)` 为 0，循环就会一直持续下去

只要有事件发生，`SDL_PollEvent(&event)` 不会为 0，程序就会进入内存循环

根据逻辑来处理不同代码，当关闭 `SDL` 程序时（包含使用快捷键关闭），发生 `SDL_EVENT_QUIT` 类型事件，进入分支改变 `quit` 的值，循环就结束了
``` c
bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        }
    }
}
```

跑通事件之后希望实现一个功能，按下 `a` 键图片往左边移动，这就需要处理键盘事件

键盘事件的类型是 `SDL_EVENT_KEY_DOWN`，同时使用 `SDL_GetKeyName(event.key.key)` 获取按下的键并且 log 出来

程序运行后可以在终端看到对应的输出
``` c
bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        } else if (event.type == SDL_EVENT_KEY_DOWN) {
            SDL_Log("key down %s\n", SDL_GetKeyName(event.key.key));
        }
    }
}
```

`SDL_GetKeyName` 返回的是字符串，在 C 语言里面判断字符串相等很麻烦，比如想判断是不是按下了 `A` 键，需要处理成下面这样
``` c
const char *key = SDL_GetKeyName(event.key.key);
if (strcmp(key, "A") == 0) {
    SDL_Log("A pressed");
}
```

判断按下什么键是高频功能，所以 `SDL` 直接提供了另外的方案
``` c
if (event.key.key == SDLK_A) {
    SDL_Log("A pressed");
}
```

按下 `A` 键后，让图片往左边移动，结合上一个例子的代码，就是在原本画图的位置偏左边一点重新画图

比如第一次画在 `(50, 100)` 的位置，第二次画在 `(45, 100)` 的位置，第三次画在 `(40, 100)` 的位置

这样看起来就相当于图片在往左边移动

``` c
int speed = 5;
SDL_Surface *image = SDL_LoadPNG("paddle.png");
SDL_Rect rect = {
    .x = 50,
    .y = 150,
    .w = image->w,
    .h = image->h,
};

Uint32 color = SDL_MapSurfaceRGB(surface, 255, 255, 255);
SDL_FillSurfaceRect(surface, NULL, color);

bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        } else if (event.type == SDL_EVENT_KEY_DOWN) {
            if (event.key.key == SDLK_A) {
                SDL_Log("A pressed");
                rect.x -= speed;
            }
        }
    }

    SDL_BlitSurface(image, NULL, surface, &rect);
    SDL_UpdateWindowSurface(window);
}
```

运行之后会发现虽然图片在往左边移动，但会有残影，这是因为上一次的图片没有清除，所以看起来就有残影

在每次画图之前都重置 `surface` 就可以解决残影问题

``` c
bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        } else if (event.type == SDL_EVENT_KEY_DOWN) {
            if (event.key.key == SDLK_A) {
                SDL_Log("A pressed");
                rect.x -= speed;
            }
        }
    }

    Uint32 color = SDL_MapSurfaceRGB(surface, 255, 255, 255);
    SDL_FillSurfaceRect(surface, NULL, color);

    SDL_BlitSurface(image, NULL, surface, &rect);
    SDL_UpdateWindowSurface(window);
}
```

相当于每次循环里面做两件事情
1. 清除 `surface`
2. 在 `surface` 上面画图

接下来有一个新问题，按住 `A` 键不放的情况下，图片会一直往左边移动

但是观察移动状态，会发现其实不算流程，有种一顿一顿的感觉

其实如果看终端里的 log 情况也能发现，1s 的时间里大概触发了 10 多次按键事件

从游戏的角度来看，60fps 会流畅很多，也就是按键事件 1s 能触发 60 次，即每一帧都能触发一次按键事件，也就是状态更新放在外层循环，而不是内层循环

大方向上有两种思路可以实现这点

### 方案一

使用 `SDL_GetKeyboardState` 获取键盘状态，然后判断对应的键是否按下

图片太靠左边，没注意加刷到 `surface` 外面，所以加上往右边移动的功能，方便对比查看

``` c
bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        }
    }
    const bool *keystate = SDL_GetKeyboardState(NULL);
    if (keystate[SDL_SCANCODE_A]) {
        SDL_Log("A pressed in frame");
        rect.x -= speed;
    }
    if (keystate[SDL_SCANCODE_D]) {
        SDL_Log("D pressed in frame");
        rect.x += speed;
    }

    Uint32 color = SDL_MapSurfaceRGB(surface, 255, 255, 255);
    SDL_FillSurfaceRect(surface, NULL, color);

    SDL_BlitSurface(image, NULL, surface, &rect);
    SDL_UpdateWindowSurface(window);
}
```

### 方案二
自己维护一套按键状态，键盘按下和键盘抬起的时候分别处理状态的值

在外层循环根据按键状态执行对应的逻辑

``` c
bool keyA_pressed = false;
bool keyD_pressed = false;
bool quit = false;
while (quit == false) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_EVENT_QUIT) {
            quit = true;
        } else if (event.type == SDL_EVENT_KEY_DOWN) {
            if (event.key.key == SDLK_A) {
                keyA_pressed = true;
            }
            if (event.key.key == SDLK_D) {
                keyD_pressed = true;
            }
        } else if (event.type == SDL_EVENT_KEY_UP) {
            if (event.key.key == SDLK_A) {
                keyA_pressed = false;
            }
            if (event.key.key == SDLK_D) {
                keyD_pressed = false;
            }
        }
    }

    if (keyA_pressed) {
        SDL_Log("key a pressed");
        rect.x -= speed;
    }
    if (keyD_pressed) {
        SDL_Log("key d pressed");
        rect.x += speed;
    }

    Uint32 color = SDL_MapSurfaceRGB(surface, 255, 255, 255);
    SDL_FillSurfaceRect(surface, NULL, color);

    SDL_BlitSurface(image, NULL, surface, &rect);
    SDL_UpdateWindowSurface(window);
}
```
